## What This Is

DisposableCheck — a React SPA that lets users check if an email is from a disposable provider. It includes single/bulk checking, an API docs page, a blog, and SEO-focused content pages. The frontend calls a separate Cloudflare Workers API at `disposablecheck.irensaltali.com/api`.

## Commands

```bash
pnpm install          # Install dependencies (project uses pnpm)
pnpm dev              # Dev server on http://localhost:8080
pnpm build            # Production build (outputs to dist/)
pnpm lint             # ESLint
pnpm test             # Vitest (run once)
pnpm test:watch       # Vitest (watch mode)
```

## Architecture

- **Vite + React 18 + TypeScript** SPA with `react-router-dom` for client-side routing
- **Styling**: Tailwind CSS + shadcn/ui component library (components in `src/components/ui/`)
- **Path alias**: `@/` maps to `src/`
- **State**: `@tanstack/react-query` for server state; local React state otherwise
- **Charts**: chart.js / recharts / plotly.js (used on the Research page)

### Key directories

- `src/pages/` — Route-level page components, each wrapped in `<Layout>` and `<SeoHead>`
- `src/components/` — Shared components (`Layout`, `EmailChecker`, `SeoHead`, `StatsCard`, etc.)
- `src/components/ui/` — shadcn/ui primitives (auto-generated, avoid manual edits)
- `src/content/blogPosts.ts` — All blog post data as typed objects (not markdown files). Each `BlogPost` has slug, SEO fields, sections, FAQ, and CTA. Adding a blog post means adding an entry here.
- `src/lib/seo.ts` — Breadcrumb building, canonical URL helpers, route label registry
- `src/lib/mockData.ts` — Client-side email validation helper and fallback domain list

### Routing

Routes are defined in `App.tsx`. Blog posts use `/blog/:slug` with slugs from `blogPosts.ts`. The sitemap plugin in `vite.config.ts` must list all routes including dynamic blog routes (imported via `blogSitemapRoutes`).

### SEO system

Every page must use `<SeoHead>` which imperatively manages `<head>` tags (title, meta, canonical, Open Graph, JSON-LD). Rules are codified in `SEO_RULES.md`:

- No `meta keywords` anywhere
- Self-referencing canonical on every route
- Exactly one `<h1>` per page
- `BreadcrumbList` JSON-LD on every page (auto-generated by `SeoHead`)
- `Organization` JSON-LD on the home page
- `Article` + `FAQPage` JSON-LD on blog/article pages when FAQ content exists

### API integration

The frontend calls the production API directly (`https://disposablecheck.irensaltali.com/api/v1/...`). There is no local API mock server. The `EmailChecker` component debounces input and calls `/api/v1/check`. Stats come from `/api/v1/stats`.

## Testing

Tests live in `src/test/` and use Vitest with jsdom + React Testing Library. The setup file (`src/test/setup.ts`) polyfills `matchMedia`.
